Form

L'unica personalizzazione fatta per i form nel file lomb_ver.css riguarda il tag legend. Associando la classe legend-lispa, si ottiene di far apparire il testo compreso nel tag, graficamente simile alle label dei campi di immissione (si veda nel template la dicitura Sesso relativa ai radiobutton Maschio e Femmina). 

legend.lispa-form {
   margin-bottom: 0;
   border-bottom: none;
   font-size: inherit;
   font-weight: bold;
}

Per tutto il resto si utilizzano le normali classi di bootstrap.
La strutturazione della videata va quindi eseguita utilizzando il Grid system del framework, che permette di ottenere un layout responsivo.
Nel template sono presentati due esempi dello stesso form, uno con le etichette poste verticalmente sopra i campi di immissione, l'altro con le etichette poste a sinistra dei campi.

Campi data

Al momento il type="date" per i campi di input non è correttamente supportato da tutti i browser.
Per ora risulta più opportuno non specificare tale tipo dati e utilizzare un plug-in javascript per il controllo della data e l'emissione di un calendario da cui selezionare la data.
Il plug-in scelto è bootstrap-datepicker, che ha molte opzioni di personalizzazione ed è navigabile anche via tastiera.

Per utilizzarlo è necessario aggiungere in fondo alla pagina il richiamo allo script relativo ed alla personalizzazione per la lingua italiana. 

<script src="js/bootstrap-datepicker.min.js"></script>
<script src="js/locales/bootstrap-datepicker.it.min.js"></script>

Inoltre deve essere inserito il link al suo css specifico. 

<!-- Custom styles for this template -->
<link href="css/bootstrap-datepicker3.min.css" rel="stylesheet">
<link href="css/lomb_ver.css" rel="stylesheet">

Per ogni campo data va specificata l'associazione della funzione datepicker, con le relative opzioni di configurazione del calendario. 

<script>
$("#data_rilascio").datepicker({
  format: "dd/mm/yyyy", weekStart: 1, language: "it",
  todayHighlight: true, autoclose: true
});
</script>

Per l'elenco delle opzioni disponibili e di tutte le altre possibilità di configurazione, si rimanda alla documentazione del plug-in On line è anche disponibile una demo che funge anche da configuratore interattivo.

Campi importo (currency)

Per la formattazione dei campi di input contenti importi, è possibile utilizzare il plug-in autoNumeric, che ha molte opzioni di personalizzazione, ma obbliga a specificare il type="text" sul campo.

Per utilizzarlo è necessario aggiungere in fondo alla pagina il richiamo allo script relativo. 

<script src="js/autoNumeric-min.js"></script>

Per ogni campo data va specificata l'associazione della funzione autoNumeric, con le relative opzioni di configurazione dei separatori delle migliaia, dei decimali e della valuta. 

<script>$('#importo_in_euro').autoNumeric('init', {aSep: '.', aDec: ',', aSign: '€ '});</script>

Per l'elenco delle opzioni disponibili e di tutte le altre possibilità di configurazione, si rimanda alla documentazione del plug-in On line è anche disponibile un tutorial.

Accessibilità

In questi template per i form non sono stati inseriti attributi o tag specifici per l'accessibilità.
Indispensabile è legare la giusta label ad ogni campo di input, tramite l'attributo for per il tag <label> ed il corrispondente attributo id dal tag <input>. 
<label class="control-label" for="cognome">Cognome</label>
<input id="cognome" class="form-control" type="text"/>

Leggendo la pagina con alcuni screen-reader non si sono riscontrate particolari difficoltà ad eccezione dei campi con attributo type="date", di cui sopra.
Per questi l'importante è che sia possibile digitare la data direttamente da tastiera, ed ottenere un eventuale messaggio di errore se errata.
Non deve avvenire automaticamente la comparsa di un calendario.

L'accessibilità effettiva della pagina dipende molto anche dalla disposizione dei campi a video.
E' quindi necessario, dopo aver passato l'analisi con il Web Accessibility Checker, eseguire un test con uno screen reader sulla pagina creata.
