Introduzione

I nuovi template sono basati sul framework bootstrap, sono responsivi e accessibili.
Il loro scopo è facilitare lo sviluppo di pagine HTML per applicativi, che rispettino le linee guida della Regione Lombardia.
I template consistono in una serie di file HTML di esempio, di CSS allegati e di immagini grafiche per sfondi e icone.
Il loro scopo è facilitare lo sviluppo di pagine HTML per applicativi, che rispettino le linee guida, in proposito, della Regione Lombardia.
I file HTML sono dei semilavorati adatti ad essere copiati e personalizzati per i propri scopi.
I file CSS, in particolare quello generale (style_lombardia.css) invece, non devono essere modificati. Ad essi è possibile aggiungere altri fogli addittivi per le proprie esigenze specifiche.

Criteri generali

Nomenclatura delle classi
I nomi di classe attribuiti ai tag html, cui fanno riferimento i CSS, cercano, quando possibile di corripondere ai termini usati nelle linee guida.
Validazione
Tutte le pagine dei template sono state validate XHTML 1.0 e CSS 2.1
Compatibilità
La resa dei template è stata provata sui seguenti browser:
  • Internet Explorer, versioni 8 e 9
  • Firefox 19
  • Safari 6.0
  • Chrome 25
Layout
Per determinare il layout le pagine sono stati utilizzati solo tag <div> e <span>.
Il tag <table> è stato utilizzato solo per il disegno di reali tabelle.
Link
Pressochè tutti i link dei template riportano al documento stesso: href="#", tranne quello sull'immagine del logo della regione Lombardia, che riporta al sito istituzionale della regione.
Inoltre nei form è impostato method="get" per poter, premendo i pulsanti, vedere sulla barra dell'indirizzo quali dati vengono inviati al server.

I file HTML

Sono di due tipi:
Componenti singoli
Sono comodi per il copia-incolla delle singole parti della videata da comporre e per capire più facilmente la struttura del componente stesso.
Composti
Creati assemblando i componenti singoli in vario modo, come esempi di pagine complete di applicativi.
Sono utili per vedere come nidificare i componenti ed ottenere una pagina completa.

I file CSS

Si trovano nella directory css/ e sono così organizzati:
stile_lombardia.css
E' il file principale. Deve essere sempre specificato.
Contiene la definizione delle caratteristiche di tutti gli elementi dei template nella declinazione cromatica del tema originale.
file delle declinazioni cromatiche
  1. azzurro.css
  2. grigio.css
  3. ocra.css
  4. viola.css
Sono addittivi rispetto a quello principale. Ne va specificato uno, solo se si intende utilizzare una declinazione cromatica diversa dal tema originale.
Nei template HTML sono inseriti commentati, nella posizione che devono occupare se attivati.
directory lombardia-theme
Tema grafico dei widget realizzati con jQueryUi: contiene sia fogli di stile che immagini.
Per l'uso si veda, ad esempio, la documentazione del calendario.

Le dichiarazioni di stile sono molto specifiche sugli elementi: difficilmente sono applicate a tutti i tag di un certo tipo o a una classe genericamente, ma sono limitate ad una classe all'interno di uno o più tag nidificati.
Questa scelta è motivata dal voler evitare conflitti con eventuali altri fogli CSS specifici degli applicativi.
Ha lo svantaggio di non consentire stravolgimenti nella struttura dell'HTML dei componenti rispetto a quanto previsto.
Poichè i componenti realizzati sono però o parti a struttura fissa (testata, piede, titolo pagina, area utente, menù) o contenitori generali (tabella, area contenuti, tabber), e pensati per degli applicativi, e non per delle pagine grafiche ad effetto per dei portali di tipo comunicativo, non dovrebbero risultare troppo costrittivi per gli sviluppatori.
Un punto critico potrebbe risultare quello dell'evoluzione nel tempo di tali template. Si suggerisce una attenzione al versioning dei template utilizzati e di non utilizzare template di versioni diverse per uno stesso applicativo.

Le immagini

Tutte le immagini utilizzate si trovano nella directory images/.

  • images/tagli_html
    contiene le immagini utilizzate come sfondi per la resa grafica dei bottoni, linguette, ecc.
  • images/icone
    contiene le icone, suddivise nelle sottodirectory 18/, 24/ e 32/ a seconda della dimensione.
    Nei template sono state utilizzate solo icone di dimensione 18x18 pixel.
    Anche le immagini nella sottodirectory tree/ non sono state utilizzate.

L'immagine fotografica della pagina di accoglienza è sempre in images/ ma non in una sottodirectory, assieme ad alcuni loghi utilizzati nella stessa pagina.
